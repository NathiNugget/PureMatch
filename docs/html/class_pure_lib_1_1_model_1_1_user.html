Constructor using paramters. Parameters are passed to the properties which confirm values are valid.

Parameters
useridID for the user. Usually one is provided from the database, but when not instantiating from the database, just use 1
nameName for the user. This is the name shown to other users. Full name of the user is expected in this field
usernameUsername is used for login and not shown to other users
passwordPassword is used for login and not shown to other users
phonenumberPhonenumber is only shown on "my profile" page. Expected value is 8 digits
emailEmail is only shown on "my profile" page. Expected form is of usual email-form
cardnumberCardnumber is only used for profile-creation. Expected value is a string with 16 digits
cardcvcCVC is only used for profile-creation. Expected value is a string with 3 digits
cardexpmonthExpiration date is only used for profile-creation. Expected value is a string with 2 digits
expyearExpiration year is only used for profile-creation. Expected value is a string with 2 digits
subscriptionSubscription is either an integer value or an enumeration of type SubscriptionsEnum. Expected values are within the enum
levelLevel is which level the user currently has. The value is expected to be within enum values while 3 is reserved for when the user is not in matching-mode
